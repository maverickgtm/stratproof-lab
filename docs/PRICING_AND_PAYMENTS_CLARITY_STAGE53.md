# Stage 53 — Pricing and Payments Clarity

## Monetization direction

Internal product principle:

> Card-first for trust. Crypto-ready for global traders.

|---|---:|---|

## Planned payment phases

### Phase 1 — Public GitHub

- Publish Community Edition.
- Collect feedback.
- Add waitlist / early access interest later.
- No payment integration in the public repo.

### Phase 2 — Early Access

Possible tools:

- Gumroad.
- Lemon Squeezy.
- Paddle.
- PayPal invoice.
- Manual crypto/stablecoin invoice for approved users.

### Phase 3 — Subscription and licensing

Possible tools:

- Stripe Billing.
- Lemon Squeezy / Paddle subscription.
- License key generation.
- Feature gating.

### Phase 4 — Crypto checkout / Enterprise

Possible tools or processes:

- BitPay.
- Coinbase Commerce / Coinbase business tooling, depending on availability.
- NOWPayments.
- BTCPay Server.
- Enterprise invoice, bank transfer, or stablecoin invoice.

## Repository safety

Never commit:

- payment API keys,
- webhook secrets,
- wallet private keys,
- customer data,
- license signing keys,
- real checkout credentials.

## Public promise

StratProof sells evidence workflows, not guaranteed returns.

The public repo should not promise:

- promises of profit,
- trading signals,
- managed accounts,
- execution,
- financial advice.
