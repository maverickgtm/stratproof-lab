# Stage 47 — Side Comparison Pro Preview

This stage adds a visible **LONG + SHORT comparison** option to the Local Workbench as a Pro preview.

Community behavior:
- Users can run LONG and SHORT one at a time.
- Selecting LONG + SHORT shows a Pro preview notice and keeps the current side.
- No paid checkout or license validation is connected.

Why this matters:
- Traders naturally ask whether the same formula works better as LONG or SHORT.
- The Community edition demonstrates the workflow without enabling the full batch side-comparison engine.
- Pro/Team can later unlock automated side-by-side reports, ranking, and side bias diagnostics.

Why LONG and SHORT can differ:
- Direction changes the TP/SL path.
- Trend filters can favor one direction.
- RSI/pullback logic may create many LONG cases in falling markets or fewer SHORT cases in weak rebounds.
- Sample size can differ across side, symbol, session, and timeframe.
- A good audit treats LONG and SHORT as separate hypotheses.
